# Fitin60

An offline-first Android app for a 60-day body transformation plan. Bring a plan
generated by ChatGPT or Claude, paste it in, and the app turns it into 60 daily
missions covering sleep, eating, and exercise — with weekly weight + photo check-ins.

## Tech

- **Kotlin + Jetpack Compose** (Material 3)
- **Room** (local persistence, no cloud)
- **MVVM** with a `Repository` layer
- **Compose Navigation**
- **kotlinx.serialization** for JSON parsing, with a markdown fallback parser
- **Android Photo Picker** for body photos (no storage permissions needed)
- **Coil** for displaying photos

Min SDK 26, Target SDK 35.

## Build & run

1. **Install Android Studio Koala (2024.1.1)** or newer, with the Android SDK
   34/35 installed.
2. Clone or extract this project at `D:\Fitin60` (or anywhere else).
3. Open the folder in Android Studio. It will index, then sync Gradle.
4. Let Android Studio download Gradle 8.7+ if prompted (the project uses AGP 8.5.2).
5. Build the app with the green ▶ button on the `app` configuration, or:
   ```
   ./gradlew :app:assembleDebug
   ```
   On Windows PowerShell:
   ```
   .\gradlew.bat :app:assembleDebug
   ```
6. Install on a device/emulator:
   ```
   ./gradlew :app:installDebug
   ```

> The repo does not include the Gradle wrapper binary (`gradle-wrapper.jar`).
> When you open the project in Android Studio it will regenerate the wrapper
> automatically on first sync. Alternatively, run `gradle wrapper` from any
> local Gradle installation.

## Sample import plan

A working sample is bundled (the "Use sample plan" button on Onboarding).
You can also paste this minimal JSON into the Import screen:

```json
{
  "programName": "Fit in 60",
  "days": [
    {
      "day": 1,
      "sleep": "Sleep by 10:30 PM, 7.5 hours minimum.",
      "meals": [
        "Breakfast: 3 eggs, oats with berries, black coffee.",
        "Lunch: Grilled chicken, brown rice, salad.",
        "Snack: Apple + 10 almonds.",
        "Dinner: Fish, roasted veg."
      ],
      "workout": [
        "5 min brisk walk warm-up",
        "Bodyweight squats 3 x 12",
        "Push-ups 3 x 8",
        "Plank 3 x 20s",
        "20 min walk cool-down"
      ],
      "notes": "Easy start - build the habit."
    },
    {
      "day": 2,
      "sleep": "Same as Day 1.",
      "meals": ["Breakfast: Greek yogurt + fruit", "Lunch: Tofu stir-fry + quinoa", "Dinner: Dal + roti + veg"],
      "workout": ["20 min jog", "Mobility 10 min"],
      "notes": ""
    }
  ]
}
```

If any days between 1 and 60 are missing from your JSON, the parser repeats the
previous day's plan to fill the gaps so you always end up with exactly 60 days.

### Markdown fallback

You can also paste plain markdown like:

```
# Day 1
Sleep: 10:30 PM, 7.5h
Meals:
- Breakfast: oats + eggs
- Lunch: chicken + rice
- Dinner: fish + veg
Workout:
- 30 min walk
- 3 x 12 squats
Notes: Easy start

# Day 2
Sleep: Same as Day 1
Meals:
- Yogurt + fruit
Workout:
- Mobility 10 min
```

The parser is lenient with case, list bullets (`-`, `*`, `•`), and section
labels (`Sleep`, `Meals`, `Food`, `Eating`, `Workout`, `Exercise`, `Notes`).

If neither format parses, the Import screen shows a friendly error message and
keeps the textbox populated so you can edit and retry.

## Assumptions

- **Day counter** is derived from the program's `startedAtMillis` rather than
  by user check-in. Day 1 = the calendar day you start the program; Day 2 = the
  next calendar day, regardless of whether you marked yesterday complete.
  Days are clamped to `[1, 60]`.
- **Weekly check-in cadence** uses week `((day - 1) / 7) + 1` and is clamped to
  `[1, 8]`. You can log any of the 8 weeks at any time, not just on the boundary.
- **Photos** picked from the Android Photo Picker are copied into the app's
  internal storage (`filesDir/checkin_photos/`) so they survive even if the
  original gallery item is deleted. They are wiped on program reset.
- **JSON parsing** is lenient (`ignoreUnknownKeys = true`, `isLenient = true`,
  `coerceInputValues = true`) so plans generated by LLMs that include extra
  fields still import cleanly.
- **Missing days** in an imported plan are filled by repeating the most recent
  known day, ensuring the user always has 60 days of content.
- **No background work / no notifications**: the "next check-in" reminder is
  in-app only. A WorkManager-driven reminder is out of MVP scope.
- **No accounts, no cloud, no analytics, no payments.** All data lives in
  `fitin60.db` and `filesDir/checkin_photos/` on-device only.
- **Theme** follows the system light/dark setting. Dynamic color is intentionally
  disabled to keep the brand palette consistent across devices.

## Known limits (intentional, kept out of MVP)

- No editing of individual days after import (you reset the whole program instead).
- No multi-program history; the latest program replaces the previous one.
- No CSV/PDF export.
- No rest-day / cardio-zone classification beyond what the imported plan says.
